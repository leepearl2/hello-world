0-2 개인정보·민감정보 처리 기준

현재 개인정보 및 민감정보를 항목 단위로 정리한 별도의 분류 기준 문서는 운영하고 있지 않습니다. 개인정보 여부는 「개인정보 보호법」상 정의 기준을 적용하여 판단합니다. 특정 항목이 단독으로는 식별성이 없더라도 다른 정보와 결합될 경우 개인을 식별할 수 있으므로, PoC 진행 시에도 문서에 포함되거나 함께 처리되는 정보의 구성에 따라 법적 정의를 기준으로 판단하는 방식이 적절합니다.

영업지원팀

시나리오 1-1 견적서·계약서 검토 프로세스

AS-IS 현재 방식

  1. F&B 영업팀이 견적서 작성 → 메일로 영업지원에 검토 요청
  2. 영업지원이 파일 다운로드 → 수정 → 재첨부 회신
  3. 수정 사항 있으면 다시 메일 핑퐁 반복
  4. 최종본 혼선 ("견적서_수정_최종_v3_진짜.xlsx")

TO-BE PoC 방식

  1. F&B 영업팀이 #contract-review에 Docs/Sheets 링크 + 요청 사항 게시
  2. 영업지원팀이 해당 문서에서 직접 제안 모드로 수정
  3. 코멘트로 질의/논의 → 스레드에서 빠른 확인
  4. 영업팀이 제안 승인/거절 → 확정
  5. 확정본은 Drive 공유 폴더에 자동 보관 (버전 이력 자동)
🔒

보안 통제

계약서·견적서는 2등급(부서비) 문서. #contract-review는 비공개 채널로 운영. Drive 공유 범위를 영업지원+해당 영업팀으로 제한. 거래처 담당자 연락처 포함 시 Slack 채널에 직접 게시 금지 → Drive 링크로 대체.

시나리오 1-2 주간 매출 실적 리포팅

AS-IS

  1. 각 영업 채널에서 엑셀 파일로 실적 취합
  2. 영업지원에서 수작업 합산 → 보고용 엑셀 작성
  3. 메일로 전략기획/CMO전략에 공유

🎯 PoC Go/No-Go 판단 기준

✅ Go 조건

  • 주요 업무 효율화 지표 3개 이상 목표 달성
  • 보안 인시던트 0건 (중대) 유지
  • 사용자 만족도 설문 4점 이상 / 5점
  • 전사 공지 인지율 80% 이상
  • GWS ↔ 기존 시스템 호환성 이슈 해결 완료

⛔ No-Go 조건

  • 1등급 문서 GWS 유입 사고 발생
  • 개인정보 유출 사고 발생
  • 주요 업무 흐름 단절 (시스템 장애 등)
  • 사용자 거부감으로 채택률 50% 미만
  • 보안 정책 미준수 반복 (3회 이상/주)
